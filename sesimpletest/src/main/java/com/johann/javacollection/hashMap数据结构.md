
HashMap是一个由key-value键值对组成的集合。

数据结构

hashMap外层是一个数组，初始化map的时候，可以设定数组的容量，如果不设置，默认是 16。
数组中的每个位置用于存储entry键值对，称之为“桶”。桶的内部数据结构可能是链表，可能是红黑树(1.8新增)。

put方法分析

在初始化一个HashMap的时候，这个map内部数组中的每个位置都是null，当我们新增一个元素 .add("johann","21")。此时，会先计算这个元素key的hashcode值，
根据这个hashcode值，计算出此时这个元素应该放在数组的哪个位置（index的计算方式稍后介绍），假设 index = 2 ，我们就会把这个元素放到 外层数组[2]这个位置。
同理， .add("zhao","28")，计算出index = 5，则把这个元素放到 外层数组[5] 这个位置。
此时，我们会想，当我们在不断新增元素的时候，出现index冲突的情况时，hashmap是如何操作的？
假设我们又新增了一个元素 .add("lee","23"),计算得到index = 2。而在index=2的位置上，已经有一个元素("johann","21")了，
此时，hashmap会hashmap会把新加入的这个元素插入到旧元素的前面（头插法），组合成一个链表或红黑树。
